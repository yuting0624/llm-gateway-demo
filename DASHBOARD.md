# Looker Studio ダッシュボード設定ガイド

## BigQuery Federation ビューとの接続

BigQuery Federation を使い、Cloud SQL のデータをリアルタイムでクエリします。
事前に `daily_model_spend` / `daily_user_spend` ビューを作成しておいてください（手順は article.md 参照）。

1. [Looker Studio](https://lookerstudio.google.com/) を開く
2. 「空のレポート」を作成
3. データソースの追加 → BigQuery を選択
4. プロジェクト: `<your-project-id>` → データセット: `llm_gateway` → ビュー: `daily_model_spend` を追加
5. 同様に `daily_user_spend` も追加

## 推奨チャート

### 1. モデル別コスト（円グラフ）
- **データソース**: `daily_model_spend`
- **ディメンション**: `model`
- **指標**: `SUM(total_spend)`
- **用途**: Claude vs Gemini のコスト比率を一目で把握

### 2. ユーザー別コスト（横棒グラフ）
- **データソース**: `daily_user_spend`
- **ディメンション**: `user_id`
- **指標**: `SUM(total_spend)`
- **並べ替え**: コスト降順
- **用途**: 誰が一番使っているか

### 3. リクエスト数の時系列（折れ線グラフ）
- **データソース**: `daily_model_spend`
- **ディメンション**: `date`
- **指標**: `SUM(request_count)`
- **内訳ディメンション**: `model`
- **用途**: 利用トレンド、モデル別の推移

### 4. トークン消費量（スコアカード + 時系列）
- **データソース**: `daily_user_spend`
- **指標**: `SUM(prompt_tokens)`, `SUM(completion_tokens)`
- **用途**: トークン効率の分析

### 5. モデル×日次積み上げ（積み上げ棒グラフ）
- **データソース**: `daily_model_spend`
- **ディメンション**: `date`, `model`
- **指標**: `SUM(total_spend)`
- **用途**: 日次のコスト推移をモデル別に把握

## フィルター設定

以下のフィルターをダッシュボードに追加すると便利：
- **期間セレクタ**: `date` ベース
- **モデルフィルター**: `model` のドロップダウン
- **ユーザーフィルター**: `user_id` のドロップダウン
